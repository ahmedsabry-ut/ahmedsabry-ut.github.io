So after a much needed 4 month break, it was time to get back to the grind. And what better way to get back than to take the most difficult class in the degree...

![Donkey meme](/assets/images/memes/donkey.jpeg)


In all fairness, I wanted to get back to the grind, but I only wanted 1 course because I didn't want to context switch between 2 classes when I'm still getting back to it. This class was also a graduation requirement, so I needed to get rid of it eventually, and there was no way I'd balance this class with any other class.

This course was the most difficult rated class in the degree at the time, and for good reason. This course is split into 2 halves, a theoretical section and a practical section, we started off with the theoretical half of the course for the first 2 months, and honestly... that was brutal

This class had an approximate workload of 13 hours/week and a difficulty rating of 7.8/10, and I knew this going into the course. But, if I could rate each half of the semester, I'd rate the first half as a 20 hours/week workload with a difficulty of 9/10, the only reason the rating isn't the case is because the second half was SIGNIFICANTLY easier.

The first half taught the actual theory of machine learning, how we don't actually want to memorize the data, how we can prove that there are limits to the mistakes we could make, PAC learning, and basically how you can formally talk through how an algorithm can perform in a certain way with data. All very useful, but goddamn was that difficult. Scored a solid 39/60 on the midterm, only slightly above class average of 36.5/60.

The second half of the class though ... wow, night and day honestly. The 2nd half was objectively easier. MLE, Bayesian Inference, Clustering, etc. all things I'm familiar with. There were new concepts such as Expectation Maximization, Kernel Method, Neural Networks, but still, they're practical... if you can code them up, you can get along just fine. That second half brought me back to an A-
